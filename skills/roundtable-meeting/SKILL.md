# Roundtable 会议流程（发起 + 回流）

Roundtable 是本机的多智能体决策委员会（`<Roundtable 安装目录>`，服务常驻 `http://127.0.0.1:7777`，登录自启；未在线时运行项目目录下 start-server.cmd）。本技能覆盖两个动作：**发起会议**（自动整理简报、预填表单、弹出浏览器）与**导入结论**（读裁决卡回到当前工作）。

## 动作一：发起会议（用户说"开多 agent 会议"等）

### 1. 写会诊简报（你来写，不要让用户写）
从当前对话与项目状态提炼，markdown 格式：

```
# 项目会诊简报
## 项目是什么（一两句）
## 当前状态（已完成 / 进行中 / 数据或证据）
## 卡点与疑惑（用户此刻真正想解决的问题——这是全场焦点，写具体）
## 已考虑过的选项及顾虑
## 约束（时间/预算/技术栈/不可变更项）
```

简报是给外部顾问看的：自包含、不用会话内代号、关键事实带数字。议题（topic）= 用户疑惑的一句话版本。

### 2. 确保服务在运行
```powershell
# 探活；不通则后台启动（detached，不占对话）
try { Invoke-RestMethod http://127.0.0.1:7777/api/config -TimeoutSec 2 | Out-Null; "UP" } catch {
  Start-Process node -ArgumentList "src/server.js" -WorkingDirectory "<Roundtable 安装目录>" -WindowStyle Hidden
  Start-Sleep 2; "STARTED"
}
```

### 3. 存草稿并弹出浏览器
```powershell
$d = Invoke-RestMethod -Method Post -Uri http://127.0.0.1:7777/api/draft -ContentType 'application/json; charset=utf-8' -Body (@{ topic = $topic; materials = $briefing; template = 'consult'; workspace = $projectRoot } | ConvertTo-Json)
# template 按会议性质选：'consult'=项目会诊（方向/进度/疑惑类，默认）；'codev'=协作开发（要产出技术方案和代码）；'general'=通用辩论
# 模板必须随草稿携带——否则页面停在默认模板，顾问会按错误的输出结构发言（曾导致整场会输出对不上题）
# workspace = 当前项目根目录的绝对路径（讨论的是具体项目时务必带上）——参会 AI 将以该目录为只读工作区，
# 自行查阅代码/文档核实简报说法，发言带文件行号引用，避免纸上谈兵。非项目类议题（纯决策讨论）可省略。
Start-Process "http://127.0.0.1:7777/#draft=$($d.id)"
```
（Body 含中文时务必 UTF-8：可将 JSON 先写临时文件再 `-InFile`，或用 `[System.Text.Encoding]::UTF8.GetBytes` 包装。）

### 4. 告知用户
- 浏览器已打开、议题与简报已预填，选阵容后点「开始第 1 轮」
- 推荐阵容：模板「项目会诊」，辩手 **Claude vs 清月**（工程视角 × 经营者成本/边界/激励视角），仲裁 **Hermes**（第三方引擎不偏袒）；纯技术问题可换 Claude vs Codex
- 提醒：会开完后回来说"**导入会议结论**"

## 动作二：执行裁决（用户说"导入会议结论"/"执行裁决卡"，或直接粘贴裁决卡）

1. **拿到裁决内容**：用户粘贴的原文优先；否则读 `<Roundtable 安装目录>\sessions\` 下 mtime 最新目录的 `judge-card.md`（无则 `summaries/` 最新，再无则 `session.md`）
2. **逐条评估，不盲从**（顾问只看到简报，不了解全部实情）：把行动项分三类明确列给用户——直接采纳执行（说明为何无争议）/ 与项目实际冲突（说明冲突与改法）/ 需用户拍板（选项与各自代价摆清，如"遗留分歧"表里的项）
3. **经用户确认后执行**：逐项落地（改代码/写文档/做实验），能验证的都要验证（跑测试、实测）
4. **回填采纳追踪**（该场会议目录内 `outcome.md`，Roundtable 的方向判据数据，必填）：是否采纳裁决 / 采纳了哪条 / 是否落地成文件改动 / 落地耗时 / 未落地原因 / 本周期是否有会可开
5. **汇报**：做了什么、验证结果、哪些项在等用户拍板

## 注意

- 简报里不要放密钥/token（Roundtable 落盘有擦除，但源头干净更好）
- 一场会每轮消耗各参会 CLI 的真实调用；建议 2-3 轮后进裁决
- 服务重启后进行中的会话会归档，但可在侧边栏点「恢复」继续

## 动作三：成果回流与汇总（会后并行执行的收口）

裁决卡的行动项常被分派给多个 AI（如 Claude 做测试、Codex 做探索）并行执行。约定：

**成果回流**（每个执行方做完自己那份后）：把成果报告写入该场会议目录 `sessions/<dir>/results/claude-<任务名>.md`（Codex 侧写 codex- 前缀），固定结构：
```
# 成果报告：<任务名>
- 分派来源：<会议议题>裁决卡第 N 项
- 做了什么：
- 验证结果：（测试/实测输出摘要）
- 产出物：（文件路径/提交号）
- 遗留问题：
```

**汇总**（用户说"汇总会议成果"）：
1. 找到目标会议目录（最新或用户指明），读 `results/` 下全部报告 + `outcome.md`
2. 产出整合报告：互补成果合并叙述 / 冲突结论并排对比（标出分歧点，交用户裁断或建议再开一场会）/ 重复工作去重标注
3. 把落地情况回填 outcome.md（落地耗时、产出物清单）
4. 若两边结论实质冲突且影响方向，建议以两份报告为背景材料再开一场 Roundtable 复盘会
