# 功能候选:裁决卡投放到任意工作台

状态:候选(未排期)。来源:2026-07-21 用户找不到回流入口的真实困惑。

## 一句话

「回流」目前只对**从工作台升格而来**的会议可用(按 origin 回到来源台);无来源的会议(草稿直建/手建)裁决出来后,只能手动复制。应允许把裁决卡投放到**用户选定的任意工作台**(或就地新建一个)。

## 最小切法

- 裁决卡按钮区:origin 存在 → 现行「回流到来源工作台」;origin 为空 → 「投放到工作台…」弹选择列表(活动+归档工作台,或"新建");
- 复用现有 flowback 的 note() 注入路径,归档台走 resume-from-disk(代码已有,server flowback 分支就是这么做的);
- 投放消息头注明来源会议(现有格式已含)。

## 边界

- 语言:投放消息按目标工作台的会话语言渲染(机制已有);
- 不做多目标群发(一次一个,保持"谁接活谁负责"的清晰度)。
