# Demo 录屏分镜（照着点 + 念，约 60–90 秒）

目标：让没用过的人 60 秒看懂「多个 AI 一起聊、能互相辩、能动手改文件且改动要你批准」。
录成 GIF 或短视频放进 README 顶部。**全程用真实调用**——这正是卖点，别用假数据。

准备：
- 服务已跑（双击 `open-roundtable.cmd`），页面默认停在建台页；若正处于其他会话，可点侧栏唯一入口「+ 开始」回到这里。
- 侧边栏建议先清干净，或新建一个干净工作台，避免露出无关历史。
- 挂载目录选一个**小的真 git 仓库**（改动小、diff 清爽）。别选敏感项目——录屏会曝光文件名。
- 窗口调到干净尺寸，字号别太小。

---

## 镜头 1 · 建立工作台（~10s）
- 从默认建台页开始；若需要重置，**点**左侧「+ 开始」
- 在选人区只**勾参与框**选中 Claude + Codex；角色下拉保持「提案者」，「仲裁」是独立职责勾选，不算参与框
- **项目目录**填一个小 git 仓库路径
- **点**「开聊」
- 旁白：「把几个 AI CLI 拉进一个群聊——花的是你已经付的订阅，没有额外 API 费。」

## 镜头 2 · 多模型 + 互聊（~20s）
- **输入** 一个有讨论空间的小问题，例如「这个项目该先加测试还是先写文档？各说一句」→ 勾两个模型 → 发送
- 等两个模型各回一句（真实调用，稍等几秒）
- **点**「让他们讨论 × 2 轮」
- 等模型互相接话、点名反驳
- 旁白：「不只是各答各的——让它们互相辩，聊不出新东西会自动停。」

## 镜头 3 · 右栏指派修改（~25s）
- 在右侧「变更」栏的**执行者**下拉选择 Codex
- 在右侧**任务框**输入明确的小任务，例如「在 README 末尾加一行项目一句话简介」
- **点**「指派 Codex 修改」；不再复用左侧聊天输入框，也没有旧「动手」按钮
- 左侧聊天流显示修改过程与轻量变更引用；完整 **diff 卡片**只进入右侧 `#wbChangesList`
- 在右栏**展开**单文件 diff，给观众看一眼绿色新增行
- 旁白：「让它真的改代码——但它改的是隔离副本，你不点应用，你的项目一个字都不变。」

## 镜头 4 · 审批（~15s）
- **点** 某个文件的「应用」（或「全部应用」）
- 状态变「✓ 已应用到主工作区」
- 回终端 `git status` 或 `git diff` 快速晃一下，证明改动真落地了、且没自动提交
- 旁白：「审批过才落地，commit 还是你自己的事。改坏了？点丢弃，副本直接扔掉。」

## 收尾（~5s）
- 回到工作台全貌，**点**「就此开会」，停在已预填的正式会议表单
- 会议中的自由追问区叫「场边追问」；若继续跑到裁决，裁决卡会自动落回来源工作台时间线，无需手动回流镜头
- 旁白：「日常在工作台聊，重决策升级成正式会议——同一批 AI，两种档位。」

## 一句话文案（配 GIF 用）
> 把你的 Claude Code、Codex 拉进一个群聊：一起聊、互相辩、动手改代码，改动要你逐个批准。骑你自己的 CLI 订阅，零依赖，全在本机。

## 剪辑提示
- 模型真实调用有几秒延迟——剪掉等待，只留「发出→有结果」，节奏才紧。
- diff 卡片和「✓ 已应用」是最有说服力的两帧，各多停 1 秒。
- 全程别出现 API key、敏感路径、无关会话标题。
